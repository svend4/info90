# 05. Как убран каждый минус: полная сводная таблица

Каждая строка: форма → её исторический минус → конкретный механизм,
которым Живая Библиотека его устраняет.

## Потоковые формы

| Форма | Исторический минус | Механизм решения |
|---|---|---|
| **Чат** | Знания тонут, ничего не находится | Поток не хранит знания по определению; кнопка «в канон» + ИИ-дистилляция; метрика конверсии чата в канон |
| **Чат** | Синхронность: надо быть онлайн | Важное кристаллизуется и доставляется дайджестом — пропустил чат, не пропустил знание |
| **Канал** | Односторонность, нет обратной связи | Каждая доставка ссылается на карточку с тредами и правками — читатель в одном клике от участия |
| **Микроблог** | Лимит убивает нюанс, скандальность | Короткой формы для знаний просто нет: поток — для жизни, нюанс — в карточках |
| **Соцсети** | Эхо-камеры, engagement-алгоритм | Рекомендательного алгоритма нет; хронологические подписки; типизированные связи «противоречит» видны на карточке |
| **Соцсети** | Архив принадлежит платформе | Git + Markdown, экспорт, зеркала, федерация |
| **Stories** | Нулевой архив | Эфемерность допустима только в потоке, и то с дистилляцией |

## Архивные формы

| Форма | Исторический минус | Механизм решения |
|---|---|---|
| **Форум** | Обсуждение оторвано от предмета, итог не найти | Треды привязаны к объектам; у треда обязательный «итог в канон» |
| **Форум** | Снобизм старожилов, «читай FAQ» | Песочница + наставники; неполный ответ помечается ИИ, который подставляет ссылку |
| **Форум** | Умирает — и архив исчезает | Открытое хранилище, экспорт, зеркала |
| **Вики** | Страницы «ничьи», устаревают незаметно | Обязательный `keeper`, даты проверки, статусы свежести, фон-сканер устаревания |
| **Вики** | Войны правок | Версии + обязательный тред перед откатами + арбитры по процедуре |
| **Вики** | Порог входа, «мой текст перепишут» | Вклад = принятая правка с именем в истории; соавторство видно навсегда |
| **Q&A** | Окаменение «лучшего ответа» | Принятый ответ — лишь черновик карточки; дальше вечные ревизии |
| **Q&A** | Токсичность к новичкам, полиция правок | Репутация только за вклад; наставничество поощряется; ревью обязано объяснять, как исправить |
| **База знаний** | Пишется «сверху», не отвечает на реальные вопросы | Канон растёт из реальных вопросов (слой 5) и чатов (слой 3) снизу вверх |
| **Каталог** | Не масштабируется ручным трудом | Рубрики ведут хранители + ИИ-автосборка; каталог — представление канона, а не отдельный труд |
| **Каталог** | Ссылки гниют (link rot) | Еженедельная проверка ссылок ИИ, поиск архивных копий |
| **Портал** | «Всем подряд — ничем», раздувание | Главная — просто витрина каталога и подписок; нет попытки удержать пользователя силой |

## Рабочие формы

| Форма | Исторический минус | Механизм решения |
|---|---|---|
| **Репозитории (Git)** | Порог входа пугает непрограммистов | Git спрятан: интерфейс правок «как в Google Docs», ветки и merge под капотом |
| **Трекеры задач** | Обсуждение решения прячется в закрытом тикете | Тикеты не хранят знание: итог тикета обязан сослаться на карточку канона |
| **Совместные документы** | Хаос при 10+ редакторах, невидимые комментарии | Нет «все правят один документ»: есть правки-заявки с ревью (модель PR, а не документа) |
| **Рассылка** | Простыни цитат, перегруз ящика | Доставка — дайджестами с диффами; обсуждения не в почте, а в тредах у объекта |

## Новейшая форма

| Форма | Исторический минус | Механизм решения |
|---|---|---|
| **ИИ-диалоги** | Галлюцинации, правдоподобная ложь | RAG-only: ответ строится из цитат карточек; нет источника — нет утверждения |
| **ИИ-диалоги** | Нет устойчивой ссылки на ответ | Ответ ИИ — это набор ссылок на версии карточек; ссылаешься на карточку, а не на чат |
| **ИИ-диалоги** | Размывание авторства, сайты теряют трафик | Авторы и хранители подписаны на карточках и в каждом ответе ИИ; репутация людей — ядро системы |
| **ИИ-диалоги** | Модель устаревает целиком | Знание — в карточках с датами, ИИ сменяем; канон переживает любую модель |

## Системные болезни (не одной формы, а всех сразу)

| Болезнь | Механизм решения |
|---|---|
| Правило 90-9-1 (пишут единицы) | Порог первого вклада — 1 клик («предложить правку» к выделенной фразе); наставники; награда за наставничество |
| Масштаб убивает ручной труд | ИИ берёт рутину (связи, свежесть, черновики), люди — решения |
| Платформа-заложник | Открытые форматы, экспорт, федерация, зеркала — уход возможен всегда |
| Монетизация искажает форму | Модель Википедии/опенсорса: донаты и гранты; нет рекламы по вовлечённости — нет стимула выращивать гнев |

## Итог

Ни один механизм здесь не нов — новизна в **комбинации и дисциплине переходов**:
чат знает, что он не архив; ответ знает, что он не финален; правка знает,
что пройдёт ревью; карточка знает дату своей ревизии; власть знает, что её журнал
читают; ИИ знает, что он стажёр. Форма перестаёт гнить там, где у каждого
участника и каждого объекта есть понятная роль и следующий шаг.
