# 09. Чем Живая Библиотека отличается от существующих систем

Честное позиционирование: почти каждая деталь где-то уже существует.
Новизна — только в сборке. Разбор по реальным системам:

## 9.1. Построчное сравнение

| Система | Что у неё уже есть из нашего дизайна | Чего критически не хватает |
|---|---|---|
| **Википедия** | Канон, история, обсуждения, донатная экономика | Потокового слоя и чата; SLA ревизий и хранителей по имени; ИИ-дистилляции; низкий порог первой правки |
| **GitHub** | Версии, PR-ревью, Issues, Wiki, Discussions | Это для кода: порог входа, нет каталога знаний, нет статусов свежести, нет доставки подписчикам |
| **Stack Overflow** | Q&A, репутация, поисковая находимость | Окаменение ответов; нет конвейера ответ→карточка; токсичность к новичкам |
| **Discourse** | Отличные треды, доверие-уровни, гибкость | Треды не кристаллизуются в канон штатно (нужны ручные wiki-посты); нет хранителей областей |
| **Notion / Confluence** | Совместные документы, базы, поиск | Закрытость, нет публичной репутации и прозрачной власти, «кладбище устаревших страниц» без хранителей |
| **Obsidian / digital garden** | Граф связей, Markdown, локальность | Одиночная игра: нет ролей, ревью, доставки, Q&A-слоя |
| **Reddit** | Нишевые сообщества, голосование | Нет канона вообще: знание каждый раз заново в комментариях; произвол модераторов; зависимость от платформы |
| **Telegram (каналы+чаты)** | Доставка, скорость, живость | Нулевой архив и поиск по вселенной каналов; знания не кристаллизуются |
| **Fediverse (Mastodon)** | Федерация, анти-заложничество | Это поток, а не архив: память не входит в дизайн |
| **Kiwix / зеркала** | Офлайн-стойкость | Хранят чужой канон, не производят свой |

## 9.2. Почему Википедия сама не станет такой

- Её масштаб делает любую реформу почти невозможной (как перестроить самолёт в полёте);
- принцип «игнорируйте все правила» + консенсус анонимных — несовместим с
  персональными хранителями и SLA;
- у неё нет и не будет потокового слоя (чата) и ИИ-публикаций по идеологии.

Живая Библиотека — не «Википедия-2», а **формат для ниш, где Википедия не работает**:
практические, быстро меняющиеся, «недостаточно значимые» для энциклопедии знания
(именно они сейчас тонут в чатах).

## 9.3. Что заимствовано у какой системы (карта ДНК)

```
Википедия ──────► канон, история правок, донатная экономика, арбитраж
GitHub ─────────► правки-PR, ревью, журнал, соавторство, экспорт=clone
Stack Overflow ─► вопросы как заявки, голосование по существу
Discourse ──────► треды, уровни доверия, гибкая модерация
Yahoo! Directory ► идея курируемых рубрик (но исполнение: хранители + ИИ)
Telegram/RSS ───► push-доставка дайджестов (без алгоритмов)
Usenet/FidoNet ─► децентрализация, право на форк, нетикет-онбординг
RAG-паттерн ────► ИИ, отвечающий только с источниками
```

## 9.4. Одна фраза позиционирования

> Википедия хранит то, что уже стабильно. Живая Библиотека — то,
> что меняется каждый месяц и сейчас тонет в чатах:
> практическое знание сообществ с именами хранителей и датами проверки.

## 9.5. Когда НЕ нужна Живая Библиотека

- Нужен просто командный чат → берите чат, не стройте собор.
- Знание статично (справочник на годы) → достаточно обычной вики/Notion.
- Нужна скорость публикации без проверки (новости) → формат блога/канала честнее.
- Сообщество <10 человек → хватит общего документа; роли и ревью — издержки масштаба.

Форма честна, когда признаёт границы применимости — этот раздел и есть такая граница.
