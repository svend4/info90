# 16. Модель угроз Живой Библиотеки

> Документ отвечает на вопрос: **как систему будут ломать — и что в ней
> уже заложено против каждого способа.** Не «у нас есть модерация»,
> а конкретные векторы атак с конкретными механизмами защиты и их
> честными остаточными рисками. Дополняет документы 04 (доверие)
> и 07 (устав, кризисные сценарии).

Метод: для каждой угрозы — цель атакующего, вектор, защита по слоям
(техника → процесс → социум), остаточный риск по шкале Н/С/В
(низкий/средний/высокий).

## У1. Захват рубрики хранителем

**Цель:** контроль над содержимым ниши (продвижение, цензура, продажа влияния).
**Вектор:** хранитель отклоняет неудобные правки, переписывает карточки под себя.

**Защита:**
- *Техника:* вся власть в публичном журнале — каждый accept/return виден
  с обязательным комментарием (11.4); молчаливый произвол невозможен технически.
- *Процесс:* правки проходят PR-модель — хранитель не пишет в канон напрямую,
  он ревьюит; его собственные правки ревьюит второй хранитель.
- *Социум:* хранитель — срочная выборная роль (устав, отзыв 20% голосов);
  апелляция к садовникам.

**Остаточный риск: С.** Один хранитель на рубрику (текущее состояние
прототипа — жёлтая зона!) — уязвимость до появления второго хранителя.

## У2. Sybil-атака на выборы

**Цель:** избрать своего хранителя/садовника фермой аккаунтов.
**Вектор:** массовая регистрация перед выборами.

**Защита:**
- *Процесс:* право голоса — у участников со стажем и вкладом
  (принятые правки, дистилляции), а не у всех зарегистрированных.
- *Техника:* журнал вклада — объективный фильтр: у сибилов нет истории.
- *Социум:* выборы с перевесом веса тех, чей вклад в эту рубрику.

**Остаточный риск: Н–С.** Ферму с годовой историей вклада не отличить —
но её цена превышает ценность рубрики.

## У3. Флуд ИИ-черновиками

**Цель:** парализовать ревью хранителей лавиной черновиков
(атака на самый дефицитный ресурс — внимание людей).
**Вектор:** массовая дистилляция/генерация черновиков.

**Защита:**
- *Техника:* дистиллятор пишет только в inbox/ — канон неуязвим
  для флуда по построению; рейт-лимит на номинации (📌) с одного ника.
- *Процесс:* возврат черновика дёшев (одна команда + комментарий);
  серийный источник мусорных черновиков игнорируется хранителем по списку.
- *Метрика:* доля возвращённых черновиков по источнику — видна в дайджесте.

**Остаточный риск: Н.** Флуд раздражает, но не проникает в канон.

## У4. Отравление дистилляции

**Цель:** внедрить ложь в канон через тред, подсунутый дистиллятору.
**Вектор:** поддельный тред «экспертов» с согласованной ложью + фейковый источник.

**Защита:**
- *Процесс:* черновик — уровень источника C и помечен им; человек-хранитель
  обязан проверить источники по чек-листу (11.4) — дистилляция *ускоряет*
  перенос, но не *заменяет* верификацию.
- *Техника:* RAG-навигатор показывает уровень источника в футере;
  проверка ссылок сканером (link rot + живость).
- *Социум:* соавторы черновика — реальные участники треда, им публично
  приписано содержимое: репутационная цена лжи.

**Остаточный риск: С.** Изощрённая ложь с живым источником уровня B
проходит — как и везде; смягчается обязательной перепроверкой при ревизии.

## У5. Потеря инстанса (хостинг, юрисдикция, бан)

**Цель/риск:** физическое исчезновение знаний с площадки
(смерть форумов 2000-х, блокировки).
**Вектор:** отключение сервера, потеря домена, требование удаления.

**Защита:**
- *Техника:* канон = git; экспорт = `git clone` / tar.gz с sha256-манифестом;
  зеркала в разных юрисдикциях (10 §10.5); dead man's switch публикует
  снапшот при молчании инстанса.
- *Процесс:* журнал власти экспортируется вместе с каноном — новый
  инстанс наследует не только знания, но и легитимность ролей.

**Остаточный риск: Н.** Убить все зеркала + все локальные клоны — практически невозможно.

## У6. Тихое гниение (никто не атакует — все просто ушли)

**Исторически самая смертоносная «атака»**: форумы и каталоги умерли
не от взлома, а от ухода редакторов.
**Вектор:** хранители выгорают, ревизии просрочены, канон врёт по старости.

**Защита:**
- *Техника:* сканер свежести с exit-кодами в CI — гниение видно автоматически,
  заявки хранителям создаются без человека.
- *Процесс:* датированная правда — карточка не удаляется, а честно желтеет
  («устаревает»), что сохраняет доверие при деградации.
- *Социум:* ритуалы (документ 13): еженедельный дайджест-смотр,
  праздник миграции, амнистия брошенных рубрик.

**Остаточный риск: В.** Против полной потери сообщества техники нет —
есть только раннее обнаружение и достойный экспорт.

## У7. Захват доставки (алгоритмическая тирания)

**Цель:** владелец платформы меняет выдачу под рекламу/повестку.
**Вектор:** «рекомендательная» лента вместо хронологии и рубрик.

**Защита:**
- *Процесс:* в уставе запрещены алгоритмические ленты (07);
  доставка = RSS/хронология/подписки на рубрики.
- *Техника:* федерация — пользователь может читать канон с другого
  инстанса или вообще без сайта (git, RSS, tar.gz).

**Остаточный риск: Н.**

## У8. Юридическое давление на контент

**Цель:** удаление неудобных карточек через требования к площадке.
**Вектор:** жалобы, претензии правообладателей, местные законы.

**Защита:**
- *Процесс:* прозрачная процедура из устава: требование → публичная запись
  в журнале → решение садовников → при удалении карточка остаётся
  в истории git и на зеркалах (как у Википедии — но с журналом причин).
- *Техника:* зеркала в других юрисдикциях (У5).

**Остаточный риск: С.** Легитимные требования (клевета, приватность)
должны исполняться — и исполняются прозрачно; злоупотребления —
останавливаются публичностью процедуры.

## Вывод

Архитектурный принцип: **не быть неуязвимой (невозможно), а быть
дорогой для атаки и дешёвой для восстановления.** Любая успешная
атака ограничена по blast radius одной рубрикой, видна в журнале
и обратима через git. Единственная угроза с высоким остаточным
риском — не атака, а уход людей; против неё работают метрики,
ритуалы и честное старение вместо ложной вечности.
