# Прототип — Живая Библиотека (фазы 0–2)

Рабочий прототип из [документа 08](../zhivaya-biblioteka/08-ekonomika-i-zapusk.md):
**Git-репозиторий + Markdown-карточки + статический генератор + PR-модель правок + журнал власти + RSS
+ RAG-навигатор + ИИ-дистилляция + cron-сканер устаревания**.
Без зависимостей, без базы данных, без сервера — только Python 3.8+.

## Запуск

```bash
python3 build.py              # собрать сайт в dist/
python3 check_freshness.py    # сканер здоровья канона (exit 1 при красных зонах, для CI)
```

## Рабочий цикл правки (модель Pull Request)

```bash
# 1. автор редактирует копию карточки и предлагает правку
python3 propose_change.py kn-2026-0417 ./edited.md "Причина правки (обязательна)" --author pavel_d
#    → создана заявка changes/ch-NNNN.md (state: open) + запись в _ledger.log

# 2. хранитель ревьюит — комментарий обязателен и при принятии, и при возврате
python3 review_change.py accept ch-NNNN --keeper marina_k --comment "принято по чек-листу"
python3 review_change.py return ch-NNNN --keeper marina_k --comment "чего не хватает: ..."
```

Всё действия попадают в `_ledger.log` — публичный append-only журнал власти (документ 04),
который рендерится на странице `dist/ledger.html`.

## Фаза 2: ИИ-библиотекарь

Железное правило (документ 10 §10.4): **у ИИ-пайплайна нет write-доступа к канону** —
только чтение + создание черновиков-заявок. Публикует человек.

### RAG-навигатор (`ask.py`)

```bash
python3 ask.py "как устроена модерация в реддите"
python3 ask.py "что такое переобучение" --include-stale --top 3
```

Отвечает **только цитатами из карточек**: вопрос → BM25-поиск по чанкам разделов →
экстрактивный ответ, где каждый тезис привязан к `card_id §раздел` → футер с версиями
и датами проверки источников. Карточки `ustarevaet`/`trebuet-revizii` исключаются
(или включаются с янтарной пометкой). Если релевантных чанков нет — честное
«в каноне этого нет» + совет открыть вопрос. Галлюцинировать не из чего:
вне цитат навигатор молчит. (Демо-замена векторного индекса из §10.4 — BM25;
дисциплина цитирования и фильтры — как в боевой версии.)

### Дистилляция чата (`distill.py`)

```bash
python3 distill.py inbox/sample-thread.md --rubric formy/agregatory --keeper marina_k \
    --title "Ранние голоса и эффект снежного кома"
```

Тред (реплики «ник: текст») → структура «проблема → причина → решение → источники» →
**черновик** карточки в `inbox/` (статус `chernovik`, соавторы = участники треда,
URL-ы → источники) → событие `question.distilled` в журнал. Канон не трогается:
дальше черновик читает хранитель-человек. (Демо на эвристиках вместо LLM —
контракт «машина черновит, человек публикует» тот же.)

### Cron-сканер устаревания

```bash
python3 check_freshness.py --links --create-requests
```

Просроченные ревизии и сироты → заявки хранителям `changes/rq-NNNN.md` +
событие `review.requested` в журнал. **Никогда — автоправки.**
CI: [ci-freshness.yml](ci-freshness.yml) — шаблон workflow (переносится в `.github/workflows/`):
еженедельный cron (понедельник, 06:17 UTC) + каждый push в `canon/`; красные зоны
роняют сборку (exit 1), заявки коммитятся ботом.

## Структура

```
prototype/
├── build.py              # генератор сайта (канон → HTML + RSS + журнал + здоровье + черновики)
├── propose_change.py     # предложить правку (PR): diff + авто-bump версии + журнал
├── review_change.py      # ревью: accept/return, обязательный комментарий (правило 11.4)
├── check_freshness.py    # сканер здоровья (документ 13), exit-коды для CI, заявки хранителям
├── ask.py                # RAG-навигатор: ответы только цитатами канона (фаза 2)
├── distill.py            # дистилляция треда → черновик в inbox/ (фаза 2)
├── ci-freshness.yml      # шаблон GitHub Actions workflow (cron-сканер)
├── _rubrics.yml          # дерево рубрик с хранителями
├── _ledger.log           # журнал власти (append-only, публичный)
├── changes/              # заявки на правки (ch-NNNN) и заявки сканера (rq-NNNN)
├── inbox/                # черновики дистилляции + исходные треды (входной ящик)
├── canon/                # КАНОН = источник истины (Markdown + фронтматтер)
└── dist/                 # сгенерированный сайт (не редактировать вручную!)
```

## Границы (чего ещё нет)

- Ревью — локальные скрипты, не веб-форма (веб-обёртка — следующий шаг).
- Нет тредов, Q&A и чата с кнопкой «📌 в канон» как живого интерфейса.
- BM25 вместо векторных эмбеддингов; эвристики вместо LLM в дистилляции
  (интерфейсы и дисциплина — боевые, модели — подменяемые).
- Выборы садовников и вторая ниша (метрики фазы 2 из документа 08) —
  социальная часть, коду не подчиняется.
