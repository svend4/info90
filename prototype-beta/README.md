# Прототип-бета — второй инстанс Живой Библиотеки

Минимальный второй инстанс для демонстрации федерации «по-настоящему»
([документ 17](../zhivaya-biblioteka/17-protokol-federacii.md)):
**канон федеративен, власть локальна.**

У беты свой канон (`canon/`), свои рубрики (`_rubrics.yml`) и свои хранители.
Инстанс-альфа ([prototype/](../prototype/)) публикует карточки через
`federate.py` (ActivityStreams outbox по рубрикам); бета подписывается на них
скриптом `subscribe.py` из альфы:

```bash
cd ../prototype
python3 subscribe.py --outbox dist/federation/outbox-formy_katalogi.json \
    --articles-dir dist/federation --instance lib-alpha --target ../prototype-beta
```

Реплики складываются в `replicas/lib-alpha/` и подчиняются правилам протокола:

1. **Read-only.** Реплика не попадает в `canon/` беты. Правка чужой карточки
   невозможна локально — предложение уходит хранителю оригинала.
2. **Статус не наследуется.** У реплики нет статуса беты — есть `origin_*` поля
   оригинала. Доверие к реплике = решение хранителей беты подписаться, а не
   автоматика.
3. **Конфликт id → локальная версия побеждает**, событие пишется в `_sync.log`.
4. **Обновление — только по росту `origin_version`**.
5. **Удаление не распространяется.** Исчезнувшая из outbox карточка помечается
   `origin_missing_at`, но копия сохраняется — чужая цензура не стирает нашу.

## Журнал синхронизации

`replicas/lib-alpha/_sync.log` — append-only история подписки:
добавления, обновления, конфликты, исчезновения оригиналов.

## Что здесь демонстрируется

- Своя карточка [kn-2026-1001](canon/soobshchestva/forumy/kn-2026-1001-zakrytye-forumy.md) —

- Реплики рубрик `formy/katalogi` и `formy/agregatory` альфы в `replicas/lib-alpha/`.
- Сценарий «исчезновение оригинала»: при повторной синхронизации с outbox,
  где карточки уже нет, реплика сохраняется с пометкой (правило 5).

## Форк рубрики (fork.py)

Подписка даёт чужое знание дома в read-only; **форк** — «забрать рубрику
себе» ([документ 17](../zhivaya-biblioteka/17-protokol-federacii.md),
[документ 25](../zhivaya-biblioteka/25-glazami-vtorogo-instansa.md) §25.4):

```bash
cd ../prototype
python3 fork.py keygen daria_v --target ../prototype-beta
python3 fork.py fork formy/katalogi --from lib-alpha --by daria_v \
    --target ../prototype-beta --reason "забираем рубрику себе: ..."
```

Правила форка:

1. **Источник — только реплики** (форк без подписки невозможен).
2. **Происхождение в паспорте**: `forked_from {instance, card, version, url}`
   + `forked_at` — форк не скрывает, откуда знание.
3. **Локальная версия начинается с 1**: свежесть теперь забота локального
   хранителя; обновления оригинала на форкнутую карточку не распространяются.
4. **Конфликт id → отказ** (решают люди, протокол не выбирает).
5. **Реплики остаются** — история подписки не стирается.
6. **Форк подписан ключом локального хранителя** (`rubric.forked` в
   `_ledger.log`, edsig — публично проверяемо по `_keys/registry.yml`).

(ldg-0001); отказы: без подписки, без ключа, не хранитель рубрики,
повторный форк (конфликт id).
