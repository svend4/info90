# 15. Вопросы и потоки: механика границы между живым и вечным

> Документ уточняет документы 02 (архитектура), 03 (жизненный цикл)
> и 04 (доверие) в самой тонкой точке системы: **как именно устроена
> граница между потоком (чат, треды, вопросы) и каноном (карточки)**.
> Правило «поток не хранит знания» — не лозунг, а механика.
> Здесь она описана до уровня объектов и событий журнала.

## 15.1. Почему граница — главное место системы

Все исторические формы умерли или деградировали на этой границе:

- **Чаты** — границы нет: знание рождается в потоке и тонет в нём.
- **Форумы** — граница есть («прикреплённые темы»), но пересекает её
  модератор вручную, раз в год, и вечный слой быстро устаревает.
- **Википедия** — граница жёсткая: между вопросом и статьёй — пропасть,
  вопросы вообще не являются объектом системы.
- **Stack Overflow** — граница инвертирована: вопрос *есть* объект,
  но знание остаётся размазанным по тысячам дублей вопросов.

Живая Библиотека делает границу **односторонне прозрачной**:
из потока в канон — короткий формальный путь с участием человека;
из канона в поток — только ссылки. Никакой объект не живёт по обе
стороны одновременно.

## 15.2. Объект «Вопрос»

Вопрос — первоклассный объект, а не сообщение в чате. У него есть
идентификатор, статус и след в журнале.

```
---
id: q-0007
title: "Почему каталоги проиграли поисковикам?"
status: open              # open | answered | merged
asked_by: anna_p
created_at: 2026-07-19
rubric: formy/katalogi
answered_by_card: ""      # заполняется при закрытии
---
Текст вопроса произвольной формы.
```

### Жизненный цикл вопроса

```
   пользователь             система              хранитель рубрики
       │                      │                       │
       │ question.opened      │                       │
       ├─────────────────────►│ уведомление ─────────►│
       │                      │                       │ поиск по канону
       │                      │                       ├──────────┐
       │                      │                       │◄─────────┘
       │                      │         два исхода:   │
       │                      │   ┌───────────────────┴───────────────┐
       │                      │   │ ответ УЖЕ есть в каноне           │
       │  question.answered   │   │ answer_question --card kn-...     │
       │◄─────────────────────┤   │ вопрос закрывается ссылкой        │
       │                      │   ├───────────────────────────────────┤
       │                      │   │ ответа НЕТ                        │
       │                      │   │ → тред → дистилляция → черновик → │
       │                      │   │   ревью → карточка → вопрос       │
       │                      │   │   закрывается новой карточкой     │
       │                      │   └───────────────────────────────────┘
```

Два принципа цикла:

1. **Закрытый вопрос — это ссылка, а не текст.** Ответом на вопрос
   всегда является карточка. Если ответ дан словами в треде — он не
   существует для системы, пока не дистиллирован.
2. **Открытый вопрос — публичная метрика спроса.** Список open-вопросов
   рубрики — это её backlog: что канон должен, но ещё не умеет.
   Старые open-вопросы — сигнал жёлтой зоны здоровья (документ 13).

### Статус merged

Когда новый вопрос дублирует существующий, он не удаляется, а
сливается: `status: merged`, `merged_into: q-0003`. Дубликаты —
данные: частые дубли одного вопроса — сигнал, что карточку
плохо находят (проблема навигации, а не пользователя).

## 15.3. Потоковый слой

Поток — это треды внутри рубрик (и личные/свободные треды).
Механические свойства:

1. **Поток append-only и лёгкий.** Писать может любой участник
   без ревью. Реплика = строка `ник: текст` (формат совместим
   с дистиллятором — это не случайность, а контракт).
2. **Поток не индексируется как источник.** RAG-навигатор никогда
   не цитирует поток. Поиск по потоку существует, но помечен:
   «это разговор, а не канон — проверь дату и дистиллируй».
3. **У потока нет вечных объектов.** Нет «важных тем», нет закрепов.
   Единственная форма бессмертия реплики — стать частью карточки.

### Кнопка «📌 в канон»

Единственное действие власти, доступное в потоке каждому участнику:

```
thread.py pin <rubric> --keeper <хранитель>
  → distill.py превращает тред в черновик (inbox/)
  → событие thread.pinned в журнале
  → хранитель получает черновик на ревью
```

Кнопка не публикует — она **номинирует**. Право публикации остаётся
у хранителя через стандартный цикл ревью (документ 11). Это снимает
историческую болезнь форумов («модератор не заметил хороший ответ»),
не отдавая канон толпе.

## 15.4. Сигнал полезности ≠ вердикт качества

Исторический урок Reddit: голосование неизбежно превращается
в вердикт, а вердикт толпы — в наказание инакомыслия (документ 04).
Поэтому в потоке существует ровно одна реакция: **«полезно»**.

Свойства сигнала:

- **Он неотрицательный.** Кнопки «бесполезно» не существует —
  несогласие выражается репликой или правкой карточки, не минусом.
- **Он невидим как счётчик.** Число хранится в данных, но не
  показывается на реплике: нет гонки за баллами, нет эффекта
  снежного кома. Агрегат виден только в еженедельном дайджесте:
  «самые полезные реплики недели» — кандидаты на дистилляцию.
- **Он ничего не решает.** Полезность влияет ровно на одно:
  попадание в обзор недели хранителю. Она не меняет ранжирование,
  карму, права, видимость. Вердикт выносит ревью карточки, а не толпа.

Технически: `thread.py useful <rubric> "фрагмент реплики"` →
запись в `threads/_useful.log` (тот же append-only формат журнала).

## 15.5. Антисценарии и защита границы

| Атака/деградация | Механика защиты |
|---|---|
| «Поток съедает канон» — все отвечают в чате, карточки не растут | Метрика: доля open-вопросов старше 30 дней; дайджест недели показывает её хранителям |
| Флуд кнопкой 📌 (номинация мусора) | Черновик — не публикация; хранитель возвращает с обязательным комментарием (11.4); серийный флудер — санкции по уставу (07) |
| «Полезно» как скрытая карма | Нет счётчика на реплике, нет персональных сумм пользователя, только агрегат недели |
| Дистилляция приватного разговора | Потоки рубрик публичны по определению; личные сообщения вне досягаемости дистиллятора (он читает только файлы тредов) |
| Ответ словами вместо карточки | question.answered требует `--card kn-...`; словесный ответ технически не закрывает вопрос |

## 15.6. События журнала (контракт)

```
question.opened    | question=q-NNNN | rubric=... | by=ник
question.answered  | question=q-NNNN | card=kn-... | by=keeper:ник
question.merged    | question=q-NNNN | into=q-MMMM | by=keeper:ник
thread.pinned      | thread=<файл> | draft=kn-... | by=ник
```

Всё действия границы — в публичном журнале. Граница прозрачна
не только для знания, но и для власти.
