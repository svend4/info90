## Концепция в одном абзаце

Интернет перепробовал десятки форм организации информации, и каждая пала жертвой своего слабого места: чаты тонут в потоке, форумы умирают от миграции, Q&A устаревают, каталоги не масштабируются, соцсети порождают эхо-камеры, а знания человечества расползаются по неиндексируемым мессенджерам. **Живая Библиотека** — это не новая форма, а *правильно собранный гибрид*: потоковые формы (чат, лента) используются для жизни, архивные (вики, Q&A) — для памяти, механизмы GitHub (версии, ревью) — для качества, каталог — для обзора, а ИИ — как библиотекарь, который связывает всё это, но не имеет права публиковать ничего без человека.

## Три главных правила

1. **Поток не хранит знания.** Чат и лента — место разговора, а не хранилища. Всё ценное кристаллизуется в карточки канона.
2. **У каждого утверждения есть дата и хранитель.** Нет «вечных» страниц — есть страницы с ревизиями и ответственными.
3. **Человек публикует, машина помогает.** ИИ индексирует, связывает, черновит и напоминает — но решение принимает человек.
