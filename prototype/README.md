# Прототип — Живая Библиотека (фазы 0–3)

Рабочий прототип из [документа 08](../zhivaya-biblioteka/08-ekonomika-i-zapusk.md):
**Git-репозиторий + Markdown-карточки + статический генератор + PR-модель правок + журнал власти + RSS
+ RAG-навигатор + ИИ-дистилляция + cron-сканер + федерация + dead man's switch**.
Без зависимостей, без базы данных, без сервера — только Python 3.8+.

## Запуск

```bash
python3 build.py              # собрать сайт в dist/
python3 check_freshness.py    # сканер здоровья канона (exit 1 при красных зонах, для CI)
```

## Рабочий цикл правки (модель Pull Request)

```bash
# 1. автор редактирует копию карточки и предлагает правку
python3 propose_change.py kn-2026-0417 ./edited.md "Причина правки (обязательна)" --author pavel_d
#    → создана заявка changes/ch-NNNN.md (state: open) + запись в _ledger.log

# 2. хранитель ревьюит — комментарий обязателен и при принятии, и при возврате
python3 review_change.py accept ch-NNNN --keeper marina_k --comment "принято по чек-листу"
python3 review_change.py return ch-NNNN --keeper marina_k --comment "чего не хватает: ..."
```

Всё действия попадают в `_ledger.log` — публичный append-only журнал власти (документ 04),
который рендерится на странице `dist/ledger.html`.

## Фаза 2: ИИ-библиотекарь

Железное правило (документ 10 §10.4): **у ИИ-пайплайна нет write-доступа к канону** —
только чтение + создание черновиков-заявок. Публикует человек.

### RAG-навигатор (`ask.py`)

```bash
python3 ask.py "как устроена модерация в реддите"
python3 ask.py "что такое переобучение" --include-stale --top 3
```

Отвечает **только цитатами из карточек**: вопрос → BM25-поиск по чанкам разделов →
экстрактивный ответ, где каждый тезис привязан к `card_id §раздел` → футер с версиями
и датами проверки источников. Карточки `ustarevaet`/`trebuet-revizii` исключаются
(или включаются с янтарной пометкой). Если релевантных чанков нет — честное
«в каноне этого нет» + совет открыть вопрос. Галлюцинировать не из чего:
вне цитат навигатор молчит. (Демо-замена векторного индекса из §10.4 — BM25;
дисциплина цитирования и фильтры — как в боевой версии.)

### Дистилляция чата (`distill.py`)

```bash
python3 distill.py inbox/sample-thread.md --rubric formy/agregatory --keeper marina_k \
    --title "Ранние голоса и эффект снежного кома"
```

Тред (реплики «ник: текст») → структура «проблема → причина → решение → источники» →
**черновик** карточки в `inbox/` (статус `chernovik`, соавторы = участники треда,
URL-ы → источники) → событие `question.distilled` в журнал. Канон не трогается:
дальше черновик читает хранитель-человек. (Демо на эвристиках вместо LLM —
контракт «машина черновит, человек публикует» тот же.)

### Cron-сканер устаревания

```bash
python3 check_freshness.py --links --create-requests
```

Просроченные ревизии и сироты → заявки хранителям `changes/rq-NNNN.md` +
событие `review.requested` в журнал. **Никогда — автоправки.**
CI: [ci-freshness.yml](ci-freshness.yml) — шаблон workflow (переносится в `.github/workflows/`):
еженедельный cron (понедельник, 06:17 UTC) + каждый push в `canon/`; красные зоны
роняют сборку (exit 1), заявки коммитятся ботом.

## Фаза 3: федерация, зеркала, миграция

### Экспорт-зеркало (`export.py`)

```bash
python3 export.py --out /path/to/mirror
```

Полный снапшот (canon + журнал + рубрики + сайт) в `tar.gz` + `manifest.json`
с sha256 каждого файла — зеркало проверяет целостность без доверия к транспорту.
Проверено: распакованный архив побайтово совпадает с манифестом.

### Федерация (`federate.py`, документы 10 §10.5 и 17)

```bash
python3 federate.py --base https://lib.example.org
```

Каждая карточка → ActivityStreams **Article** (`attributedTo` = хранитель,
`updated` = дата проверки, source = исходный Markdown, tag = рубрики,
расширения `zh:status`/`zh:version`/`zh:reviewDue`) →
`dist/federation/`: actors.json хранителей + outbox по каждой рубрике.
Другой инстанс подписывается на outbox рубрики, как на RSS, — но с идентичностью ролей.
Правила протокола (канон федеративен, власть локальна, удаление не распространяется) —
[документ 17](../zhivaya-biblioteka/17-protokol-federacii.md).

### Dead man's switch (`deadman.py`)

```bash
python3 deadman.py --days 30          # при молчании инстанса — публикация на зеркала
python3 deadman.py --days 30 --check  # только проверка (для CI)
```

`build.py` обновляет `_heartbeat` при каждой сборке. Тишина > N дней →
автоматический экспорт и публикация tarball+манифеста на зеркала из `_mirrors.yml`
(в боевом варианте — IPFS/HTTP в разных юрисдикциях) + событие `deadman.triggered`
в журнал. **Знание не должно умереть вместе с хостингом.**
Проверено на копии: 79 дней тишины → 2/2 зеркала получили архив, журнал записан.

### Амнистия-миграция (`migrate.py`, документ 14)

```bash
python3 migrate.py inbox/migration-sample.json --keeper igor_s
```

JSON-дамп внешнего сообщества → исходники тредов сохраняются в `inbox/migration/`
(архив не теряется) → каждый тред дистиллируется в черновик (уровень источника C)
→ событие `community.migrated`. Ядро сообщества ревьюит черновики — обучение
хранителей на своём материале (недели 3–6 плана A.2). Демо: архив форума
oldweb-forum.ru → черновики kn-2026-0419 и kn-2026-0420.

## Поток, вопросы и ритуалы (документ 15)

### Вопросы (`question.py`)

```bash
python3 question.py open "Почему каталоги проиграли поиску?" --rubric formy/katalogi --author anna_p
python3 question.py answer q-0001 --card kn-2026-0007 --keeper igor_s
python3 question.py list --status open      # backlog рубрик
```

Вопрос — первоклассный объект с журнальным следом. Закрывается **только
карточкой** (`answer` требует `--card` и проверяет её существование) —
словесный ответ вопрос технически не закрывает. Open-вопросы = метрика спроса.

### Поток (`thread.py`)

```bash
python3 thread.py say formy/agregatory --nick anna_p "текст реплики"
python3 thread.py useful formy/agregatory "фрагмент реплики" --nick pavel_d
python3 thread.py pin formy/agregatory --keeper marina_k    # 📌 в канон
```

Поток append-only, реплика = «ник: текст» (формат совместим с distill.py —
контракт). «📌 в канон» — номинация, не публикация: pin → черновик → ревью.
«Полезно» — сигнал без счётчика на реплике и без отрицательных реакций:
вердикт выносит ревью, не толпа (§15.4).

### Еженедельный дайджест (`digest.py`, документ 13)

```bash
python3 digest.py    # -> digest/YYYY-Www.md
```

Ритуал хранителей: события журнала за неделю + open-вопросы (⚠️ старше
30 дней — жёлтая зона спроса) + агрегат «полезно» + зоны здоровья.

## Структура

```
prototype/
├── build.py              # генератор сайта (канон → HTML + RSS + журнал + здоровье + черновики)
├── propose_change.py     # предложить правку (PR): diff + авто-bump версии + журнал
├── review_change.py      # ревью: accept/return, обязательный комментарий (правило 11.4)
├── check_freshness.py    # сканер здоровья (документ 13), exit-коды для CI, заявки хранителям
├── ask.py                # RAG-навигатор: ответы только цитатами канона (фаза 2)
├── distill.py            # дистилляция треда → черновик в inbox/ (фаза 2)
├── export.py             # экспорт-зеркало: tar.gz снапшот + sha256-манифест (фаза 3)
├── federate.py           # ActivityPub-анонсы карточек → dist/federation/ (фаза 3)
├── deadman.py            # dead man's switch: молчание → публикация на зеркала (фаза 3)
├── migrate.py            # амнистия-миграция архива сообщества → черновики (фаза 3)
├── question.py           # вопросы как объект: open/answer/list (документ 15)
├── thread.py             # потоковый слой: say/useful/pin (📌 в канон) (документ 15)
├── digest.py             # еженедельный дайджест-смотр хранителей (документ 13)
├── ci-freshness.yml      # шаблон GitHub Actions (cron-сканер + deadman-алерт)
├── _rubrics.yml          # дерево рубрик со списками хранителей (keepers: ≥2 на рубрику)
├── _ledger.log           # журнал власти (append-only, публичный)
├── _mirrors.yml          # адреса зеркал для dead man's switch
├── _heartbeat            # признак жизни инстанса (обновляет build.py)
├── changes/              # заявки на правки (ch-NNNN) и заявки сканера (rq-NNNN)
├── questions/            # вопросы: open = backlog рубрики, answered = ссылка на карточку
├── threads/              # потоковый слой + _useful.log (сигналы полезности)
├── digest/               # еженедельные дайджесты-смотры
├── inbox/                # черновики дистилляции + исходные треды (входной ящик)
│   └── migration/        # сохранённые исходники мигрированных тредов
├── canon/                # КАНОН = источник истины (Markdown + фронтматтер)
└── dist/                 # сгенерированный сайт (не редактировать вручную!)
    ├── federation/       # ActivityPub-документы (actors, outbox, Article)
    └── export/           # manifest.json экспорта (tar.gz — в .gitignore)
```

**Возврат правки (полный круг ревью).** Заявка [ch-0002](changes/ch-0002.md)

с обязательным комментарием по чек-листу 11.4 (ldg-0016…0017). Карточка не изменилась;
проверка: `ask.py "дропаут"` честно отвечает «в каноне этого нет» — отклонённый
текст в канон не попал и навигатором не цитируется.

## Границы (чего ещё нет)

- Всё взаимодействие — CLI и файлы, не веб-формы (веб-обёртка — следующий шаг).
- Вопросы/потоки — файловые объекты; живого чат-интерфейса пока нет.
- BM25 вместо векторных эмбеддингов; эвристики вместо LLM в дистилляции
  (интерфейсы и дисциплина — боевые, модели — подменяемые).
- ActivityPub — статические JSON-документы, не живой сервер (формат обмена, не демон).
- Зеркала в `_mirrors.yml` — каталоги-приёмники вместо IPFS (контракт тот же).
- Выборы садовников и рост сообщества — социальная часть, коду не подчиняется.
