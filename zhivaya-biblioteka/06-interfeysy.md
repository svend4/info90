# 06. Интерфейсы: как выглядит каждый экран

## 6.1. Принципы UX

1. **Сложность — под капотом.** Git, версии, SLA, статусы — всё есть, но новичок
   видит кнопки «Читать», «Спросить», «Предложить правку». Мощность раскрывается
   по мере роста роли (прогрессивное раскрытие).
2. **Свежесть видна всегда.** Любой текст показывает, когда проверен и кем.
   Нет ни одного анонимного утверждения без даты.
3. **Следующий шаг — всегда один клик.** У каждого экрана есть очевидное
   «что можно сделать дальше»: из чата — в канон, из карточки — в правку,
   из правки — в ревью.

## 6.2. Главная страница (витрина портала + каталога)

```
┌──────────────────────────────────────────────────┐
│ 🔍 Поиск по Библиотеке…        [Войти] [Профиль] │
├──────────────────────────────────────────────────┤
│  РУБРИКИ (каталог)        │  СВЕЖЕЕ В МОИХ       │
│  ▸ Искусственный интеллект│  ПОДПИСКАХ (хронол.) │
│    ▸ Нейросети (412 📄)   │  • 12:40 Обновлена   │
│    ▸ Железо для ИИ (98)   │    «Типовая причина…»│
│  ▸ Разработка (1 204 📄)  │    (дифф: +2 абзаца) │
│  ▸ История сети (301 📄)  │  • 11:15 Новая карточка│
│                           │    «Модерация в Reddit»│
├──────────────────────────────────────────────────┤
│  ТРЕБУЮТ ВНИМАНИЯ         │  ВОПРОСЫ БЕЗ ОТВЕТА  │
│  ⚠ 4 карточки требуют     │  в ваших рубриках: 3 │
│    ревизии (для хранителей)│                     │
└──────────────────────────────────────────────────┘
```

Никакой бесконечной ленты. Главная отвечает на три вопроса: «что здесь есть»
(каталог), «что изменилось в моём» (подписки), «где нужна помощь» (работа).

## 6.3. Карточка знания (центральный экран)

```
┌──────────────────────────────────────────────────┐
│ ✅ Проверено 14.03.2026 · Хранитель: @marina_k   │
│ Рубрика: Формы / Агрегаторы   Версия 14 · История│
├──────────────────────────────────────────────────┤
│                                                  │
│  # Как устроена модерация в Reddit               │
│  ...текст карточки в Markdown...                 │
│  ...гиперссылки на другие карточки...            │
│                                                  │
├──────────────────────────────────────────────────┤
│ [✏️ Предложить правку] [❓ Задать вопрос]        │
│ [🔔 Подписаться на изменения]                    │
├──────────────────────────────────────────────────┤
│ СВЯЗИ: уточняет → «Голосование как модерация»    │
│        противоречит → «Миф о мудрости толпы»     │
├──────────────────────────────────────────────────┤
│ 💬 ОБСУЖДЕНИЯ (3 треда, 2 закрыты с итогом)      │
│ ❓ ВОПРОСЫ (2: 1 решён — вошёл в §3 карточки)    │
├──────────────────────────────────────────────────┤
│ История: v14 …v1 · Соавторы: @anna @pavel @igor  │
└──────────────────────────────────────────────────┘
```

Если карточка устаревает, сверху вместо зелёной плашки — янтарная:
**«⚠ Информация требует ревизии (последняя проверка: 14.03.2026)»**.
Никакой маскировки устаревшего под актуальное — главное отличие от вечно
«правдоподобных» ответов ИИ и окаменевших топ-ответов Q&A.

## 6.4. Экран правки (pull request без слова «git»)

```
┌──────────────────────────────────────────────────┐
│ Правка к карточке «Модерация в Reddit»           │
├───────────────────────┬──────────────────────────┤
│  БЫЛО (v14)           │  СТАНЕТ (ваша версия)    │
│  ...старый абзац...   │  ...ваш текст...         │
│  (подсветка удалённого)│ (подсветка добавленного)│
├──────────────────────────────────────────────────┤
│ Причина правки (обязательно):                    │
│ [ «Цены на API изменились в июне 2026»        ]  │
│ Источник: [ https://… ]  🤖 ИИ: источник валиден │
├──────────────────────────────────────────────────┤
│ 🤖 ИИ-проверка: противоречий с другими           │
│    карточками не найдено. Связанные: 2 карточки  │
│    затронуты косвенно (уведомить хранителей?).   │
├──────────────────────────────────────────────────┤
│            [Отправить на ревью хранителю]        │
└──────────────────────────────────────────────────┘
```

Пользователь редактирует как в Google Docs; система под капотом создаёт ветку,
коммит и заявку. Причина правки обязательна — это дисциплина Википедии
(«описание правки»), доведённая до обязательности.

## 6.5. Экран ревью (рабочее место хранителя)

- Очередь заявок с SLA-индикаторами (🟢 <24ч, 🟡 <72ч, 🔴 просрочено → эскалация).
- Каждая заявка: дифф, причина, источники, ИИ-справка (противоречия, стиль, факты-флаги).
- Кнопки: **Принять** / **Вернуть с комментарием** (обязательно: чего не хватает) /
  **Передать другому хранителю**.
- Правило интерфейса: «Вернуть» без текста-подсказки невозможно нажать
  (поле обязательно) — защита от «кусания новичков» на уровне дизайна.

## 6.6. Чат со встроенной дистилляцией

Обычный чат, но:
- у каждого сообщения есть «📌 в канон»;
- 🤖 ИИ периодически пишет служебное: «Похоже, в треде найдено решение
  (сообщения #45–#61). Собрать черновик карточки?»;
- закреп раздела: «Решённые вопросы не живут в чате — жмите 📌»;
- «Решено» в чате не существует: есть только «Ушло в канон → ссылка».

## 6.7. ИИ-навигатор (единственный «диалоговый» экран)

```
Вы: как reddit борется с накрутками голосов?

🤖 По карточкам Библиотеки:
• «Модерация в Reddit» (проверена 14.03.2026) — §4 описывает
  vote fuzzing и shadow-ban накрутчиков [открыть карточку]
• Связанное: «Голосование как модерация» — критика подхода [открыть]

В каноне нет данных о механизмах после июня 2026.
[❓ Открыть вопрос сообществу]
```

Ответ без ссылки архитектурно невозможен. Нет карточки — честное
«в Библиотеке этого нет» и кнопка создания заявки-вопроса.
