# 04. Доверие и модерация

## 4.1. Почему это отдельный документ

История показала: формы умирают не от плохих интерфейсов, а от краха доверия —
спам, токсичность, произвол модераторов, эхо-камеры, «вечный сентябрь».
Поэтому в Живой Библиотеке модерация — не функция, а **архитектурный слой №1**.

## 4.2. Гибридная модель модерации (все 6 исторических моделей, каждая — на своём месте)

| Историческая модель | Где провалилась | Где применена здесь |
|---|---|---|
| Централизованная (редакторы) | Каталоги: не масштабируется | Только политики верхнего уровня (совет садовников) |
| Делегированная (модераторы) | Reddit: выгорание, произвол | Хранители рубрик + публичный журнал + SLA + ротация |
| Демократическая (голосование) | Тирания большинства, снежный ком | Только сигнал полезности ответов Q&A, не решает судьбу контента |
| Репутационная (карма) | Stack Overflow: фарм, кусание новичков | Репутация = принятые правки (реальный вклад), не посты |
| Алгоритмическая | Соцсети: оптимизация гнева | Только антиспам и антифрод — **не ранжирование контента** |
| Отсутствие модерации | Usenet: спам-триумф | Нигде в каноне; допустима лишь в «курилках» потока |

**Принцип:** каждая модель сильна ровно в одном месте и смертельна в другом.
Гибрид ставит каждую на её место.

## 4.3. Репутация: как не повторить Stack Overflow

Ошибка классических систем: карма начисляется за активность (посты, ответы, голоса) —
поэтому выгодно много писать, а не много улучшать. Здесь иначе:

- **Репутация = принятый вклад.** Баллы даёт только merge: принятая правка,
  принятый ответ, принятая дистилляция из чата. Болтовня не конвертируется.
- **Нет публичной кармы пользователей.** Видна репутация **в области**:
  «хранитель рубрики X, 40 принятых правок в теме Y». Это убивает гонку чисел.
- **Помощь новичку = вклад.** Наставничество (доведение первой правки новичка
  до принятия) засчитывается наставнику — экономика поощряет учить, а не кусать.
- **Откат правки — не наказание.** Комментарий ревью обязан содержать причину
  и подсказку («чего не хватает для принятия») — правило вшито в интерфейс.

## 4.4. Прозрачная власть

Главный антидот от «reddit-2023» и форумных драм:

1. **Публичный журнал действий власти.** Каждое действие хранителя/садовника
   (отклонение правки, блокировка, ротация) — запись в открытом журнале:
   кто, что, почему, ссылка на правило.
2. **Право апелляции.** Любое решение обжалуется арбитрам; срок ответа — 7 дней;
   вердикт арбитров тоже публичен.
3. **Сроки полномочий.** Садовники и арбитры — на год, перевыборы; хранители
   подтверждаются активностью каждые 90 дней (неактивность → мягкая ротация,
   не «снятие с позором»).
4. **Правила — тоже карточки.** Регламент живёт в каноне и меняется через
   те же правки с ревью — история изменений правил видна всем.

## 4.5. Защита от исторических катастроф (карта угроз)

| Катастрофа | Год/место | Механизм защиты |
|---|---|---|
| **Вечный сентябрь** (приток новичков размывает культуру) | Usenet, 1993 | Песочница + наставники + онбординг 15 мин; права растут с принятым вкладом; метрика «вторая правка новичка» |
| **Спам-эпидемия** | Почта/блоги, 2000-е | Алгоритмический антиспам + стоимость первых действий (лимиты до первой принятой правки) |
| **Убийство платформы с данными** | GeoCities, 2009 | Хранилище = Git + Markdown, экспорт одной кнопкой, зеркала, федерация. Платформа может умереть — библиотека нет |
| **Бунт против смены правил** | Digg v4, 2010 | Правила — версионируемые карточки; изменения проходят публичное ревью с обсуждением |
| **Произвол платформы над сообществами** | Reddit API, 2023 | Публичный журнал власти, апелляции, сроки полномочий, выборность |
| **Окаменение ответов** | Stack Overflow | Ответ не финален: принятый ответ → черновик карточки → вечные ревизии |
| **Войны правок** | Википедия | Арбитры по процедуре, защита карточки на время спора, обязательный тред-обсуждение перед откатами |
| **Эхо-камеры** | Соцсети, 2010-е | Нет рекомендательного алгоритма вообще; подписки хронологические; карточки показывают противоречащие связи (`protivorechit`) |
| **Уход знаний в неиндексируемые чаты** | Discord-эра | Чат — внутри системы и дистиллируется; метрика «конверсия чата в канон» |
| **Галлюцинации ИИ** | 2023+ | ИИ отвечает только цитатами карточек (RAG); без источника — отвечает «в каноне этого нет» и открывает вопрос |
| **Link rot** | Весь веб | ИИ еженедельно проверяет внешние ссылки, ищет архивные копии, помечает мёртвые |
| **Выгорание мейнтейнеров** | Опенсорс | SLA-лимиты на одного хранителя, ≥2 хранителя на рубрику, ротация как норма |

## 4.6. Три железных правила ИИ

1. **Не публикует.** Любой текст ИИ — черновик до тех пор, пока человек
   с правами не нажал «принять».
2. **Не выдумывает.** Ответ без ссылки на карточку запрещён архитектурно (RAG-only).
   Нет карточки — ИИ честно открывает заявку-вопрос.
3. **Не решает споры.** ИИ констатирует конфликт («карточки A и B противоречат»)
   и готовит материалы, но разрешение — только люди.

ИИ здесь — то, чем исторически не хватало каталогам (масштаб ручного труда)
и вики (садовник), — но с исторически необходимым ограничителем: человеческим ревью.
