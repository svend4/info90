# Прототип Фазы 0 — Живая Библиотека

Рабочий минимальный прототип из [документа 08](../zhivaya-biblioteka/08-ekonomika-i-zapusk.md) (дорожная карта, фаза 0):
**Git-репозиторий + Markdown-карточки + статический генератор**. Без зависимостей, без базы данных, без сервера.

## Запуск

```bash
python3 build.py
# результат: dist/ — статический сайт, открыть dist/index.html в браузере
```

Требуется только Python 3.8+ (стандартная библиотека).

## Структура

```
prototype/
├── build.py              # генератор (~250 строк, zero-dependency)
├── _rubrics.yml          # дерево рубрик с хранителями
├── canon/                # КАНОН = источник истины (Markdown + фронтматтер)
│   ├── formy/agregatory/kn-2026-0417-moderaciya-v-reddit.md
│   ├── formy/katalogi/kn-2026-0007-katalogi.md
│   └── nejroseti/obuchenie/kn-2026-0201-pereobuchenie.md
└── dist/                 # сгенерированный сайт (не редактировать вручную!)
```

## Что демонстрирует прототип (соответствие документам)

| Документ проекта | Что реализовано здесь |
|---|---|
| 02, 10 (архитектура, хранение) | Карточка = Markdown-файл с YAML-фронтматтером в Git |
| 02 (объектная модель) | `id`, `status`, `freshness.verified_at/review_due`, `keeper`, `coauthors`, `links`, `sources` |
| 01 (принцип 3 — датированная правда) | На каждой странице: дата проверки, дедлайн ревизии, хранитель, цветной баннер статуса |
| 11 (политика источников) | У источников — дата `checked`; уровни S/A/B/C отражены в тексте карточек |
| 06 (интерфейс карточки) | Плашка статуса, метаданные, связи, источники, соавторы — как в макете |
| 10 («никакой магии в хранении») | Экспорт = `git clone`; версии = `git log`; сборка воспроизводима |

Одна из трёх карточек (`kn-2026-0007`) намеренно имеет статус `trebuet-revizii` —
видно, как выглядит «просроченная» карточка (принцип «устаревшее не маскируется»).

## Границы фазы 0 (чего здесь сознательно нет)

- Нет правок через веб (PR-обёртки) — правки делаются git-коммитами. Это фаза 1.
- Нет тредов, вопросов Q&A, чата и кнопки «📌 в канон». Фазы 1–2.
- Нет ИИ-навигатора (RAG). Фаза 2. Техпайплайн — в документе 10.
- Нет журнала власти (`_ledger.log`) — появится вместе с ролями.
